# FPGA Proof of Concept Using Tang Nano 1K

This repository contains a simple FPGA proof of concept implemented on the **Tang Nano 1K** board. The design emulates four digital probabilistic bits, or p-bits, using a pseudo-random LFSR-based noise source and a simplified update rule inspired by the Ising model and the Max-Cut optimization problem.

The purpose of this implementation is to demonstrate, at hardware level, the basic behavior of a probabilistic processor concept using an FPGA and a logic analyzer.

## Overview

The implemented circuit uses:

- **Tang Nano 1K FPGA board**
- **27 MHz onboard clock**
- **8-bit LFSR pseudo-random generator**
- **Four digital p-bit outputs**
- **Simplified Max-Cut evaluation**
- **Logic analyzer outputs for waveform observation**

The design does not implement a full physical STMJ device. Instead, it provides a digital emulation of stochastic p-bit behavior suitable for experimental validation and dissertation documentation.

## Signal Description

The FPGA design exposes internal signals to external pins so that they can be observed using a logic analyzer.

| Logic Analyzer Channel | FPGA Signal | Description |
|---|---|---|
| CH0 | `clk_slow` | Slow reference clock |
| CH1 | `update_en` | Visible update pulse |
| CH2 | `rand_bit` | Pseudo-random LFSR output |
| CH3 | `spin_0` | Digital p-bit 0 |
| CH4 | `spin_1` | Digital p-bit 1 |
| CH5 | `spin_2` | Digital p-bit 2 |
| CH6 | `spin_3` | Digital p-bit 3 |
| CH7 | `best_update` | Indicates a favorable Max-Cut configuration |

## Design Description

The circuit contains four digital p-bits represented by the register `spin[3:0]`.

A pseudo-random bit stream is generated by an 8-bit Linear Feedback Shift Register, or LFSR. During each update event, the p-bit states are modified using a combination of neighboring spin states and pseudo-random noise.

The circuit also evaluates a simple Max-Cut problem on a four-node ring:

```text
s0 -- s1
|     |
s3 -- s2
```

The evaluated edges are:

```text
s0-s1
s1-s2
s2-s3
s3-s0
```

The signal `best_update` is activated when the current spin configuration produces three or four cut edges. This condition is interpreted as a favorable low-energy configuration for the simplified Max-Cut formulation.

## How to Build the Project

1. Open **Gowin EDA**.

2. Create a new FPGA project for the Tang Nano 1K board.

3. Add the Verilog source file:

```text
src/top.v
```

4. Add the constraint file:

```text
constraints/tangnano1k.cst
```

5. Run the synthesis step.

6. Run place and route.

7. Generate the bitstream.

8. Program the Tang Nano 1K using the Gowin Programmer.

## How to Observe the Signals

Connect the logic analyzer channels to the FPGA pins according to the table below:

| Logic Analyzer Channel | Signal |
|---|---|
| CH0 | `clk_slow` |
| CH1 | `update_en` |
| CH2 | `rand_bit` |
| CH3 | `spin_0` |
| CH4 | `spin_1` |
| CH5 | `spin_2` |
| CH6 | `spin_3` |
| CH7 | `best_update` |

Also connect the logic analyzer ground to the Tang Nano 1K ground.

Use a sampling frequency high enough to observe the update pulses and p-bit transitions. The signal `clk_slow` can be used as a visual timing reference.

## Expected Behavior

After programming the FPGA, the logic analyzer should show:

- A slow reference clock on `clk_slow`;
- Periodic update pulses on `update_en`;
- A pseudo-random signal on `rand_bit`;
- Changing digital spin states on `spin_0` to `spin_3`;
- Pulses on `best_update` when a favorable Max-Cut configuration is reached.

The observed behavior demonstrates a simplified stochastic update process compatible with the conceptual operation of probabilistic Ising-inspired processors.

## Academic Context

This project was developed as a hardware proof of concept for a master's dissertation related to probabilistic processors based on the Ising model.

The FPGA implementation complements the simulation framework by showing that the basic concepts of stochastic p-bit updates, pseudo-random behavior, and simplified energy evaluation can be mapped to digital hardware.

## Important Notes

This project is intended for educational and academic validation purposes.

The implementation is a digital approximation and does not physically reproduce the analog behavior of stochastic magnetic tunnel junctions. However, it provides a useful experimental platform to visualize probabilistic state updates using a low-cost FPGA board and a logic analyzer.

## Safety Notes

- Use only 3.3 V compatible logic analyzer inputs.
- Do not connect FPGA pins directly to 5 V signals.
- Always connect the logic analyzer ground to the FPGA ground.
- Check the pinout before powering the board.
- Avoid short circuits on adjacent pins.
